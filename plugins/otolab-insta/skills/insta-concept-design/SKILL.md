# insta-concept-design｜Instagramアカウント設計 3ステップ

## このスキルについて

Instagramアカウントの発信の土台を、3つのフェーズで順番に固めるスキル。オトラボ受講生のサポートにも、運営者自身・クライアントのアカウント設計にも使う。

| フェーズ | 内容 | 元エージェント | 指示書 |
|---|---|---|---|
| Phase 1 | 自己分析インタビュー（経験・強み・価値観・ビジュアルの棚卸し） | インスタ自己分析インタビュアー | `references/phase1_self_analysis.md` |
| Phase 2 | コンセプト設計（Who→What→Howで3案提案） | 誰の悩みをどのようにどう解決するの？けんじくん | `references/phase2_concept.md` |
| Phase 3 | アカウント名・プロフィール作成（名前3案＋プロフ3案） | Instagram プロフィールメーカー | `references/phase3_profile.md` |

**各フェーズの指示書が本体。** フェーズを始める前に必ず該当の指示書を読み、そこに書かれた質問フロー・出力フォーマットを忠実に守ること。指示書は将来Webアプリに移植する前提で、それ単体で完結するシステムプロンプトとして書かれている。要約したり省略したりしない。

## 進め方

### 最初に必ずやること：Webアプリ（エディタ）を開く

スキルが発動したら、質問やヒアリングを始める前に、まず公開版のWebアプリをChromeで開く：

1. Claude in Chrome でユーザーのChromeに `https://insta-concept-web.pages.dev` を開く（Cloudflare Pagesで公開済み・サーバー起動は不要）
2. 招待コードは `（招待コードは講座内で配布）` と伝える

開いたら「エディタを開きました。招待コードは （招待コードは講座内で配布） です」とだけ伝える。受講生のデモ・受講生自身の利用はこのWebアプリで完結する。

ローカル版（コード修正の動作確認用）が必要なときだけ、preview_start でサーバー `insta-concept-web` を起動して `http://localhost:8788` を開く（`~/Documents/Claude/.claude/launch.json` に設定済み）。

チャットで進めるのは、ユーザーが「チャットでやりたい」「ファイルで成果物がほしい」と言った場合だけ。その場合は以下のフェーズ進行に従う。

### エントリーポイントの判定

ユーザーの依頼から開始フェーズを判定する：

- 「自己分析」「自分の強みを整理」「発信の土台」→ Phase 1 から
- 「コンセプト」「ジャンル選定」「ポジション」「Who What How」→ Phase 2 から
- 「アカウント名」「プロフィール」「ネーミング」→ Phase 3 から
- 「アカウント設計全部」「ゼロから立ち上げ」→ Phase 1 から3連続

Phase 2・3 から始める場合は、前フェーズの成果物（自己分析レポート／コンセプト決定内容）が必要。ユーザーに「前のレポートのファイルパスか、内容の貼り付け」を求める。過去の成果物は `~/Documents/Claude/Projects/インスタコンセプト設計/` 配下を探すと見つかることがある（まず探してから聞く）。どうしても無ければ、その場で最低限のヒアリング（対象者・ジャンル・強み）をして進めてよい。

### 成果物の保存

対象者ごとにフォルダを作って保存する：

```
~/Documents/Claude/Projects/インスタコンセプト設計/<対象者名または屋号>_<YYYYMMDD>/
├── 01_自己分析レポート.md   ← Phase 1 の出力（指示書内のフォーマット厳守）
├── 02_コンセプト3案.md      ← Phase 2 の出力（選ばれた案に「★採用」を付記）
└── 03_プロフィール案.md     ← Phase 3 の出力
```

- 各フェーズ完了時に必ずファイル保存し、パスをユーザーに伝える
- 次のフェーズはこのファイルを読み込んで開始する（ユーザーに再貼り付けさせない）
- PDF版を求められたら pdf スキルで変換する

### フェーズ間の接続

- Phase 1 完了時：「このまま Phase 2（コンセプト設計）に進みますか？」と確認してから進む
- Phase 2 完了時：3案から1つ選んでもらい、選択を `02_コンセプト3案.md` に反映してから Phase 3 へ
- Phase 3 完了時：3点セットの完成を伝え、次の一歩（リール台本づくり等の別スキル）を軽く案内

## Genspark版からの置き換えルール

元の指示書に登場するGenspark固有の機能は、次のように読み替えて実行する：

| 元の記述 | Claude Codeでのやり方 |
|---|---|
| Yahoo!知恵袋で調べる（Phase 2） | WebSearchで「site:chiebukuro.yahoo.co.jp <悩みキーワード>」等を検索 |
| プロフィールのスクリーンショット提出 | 画像ファイルのパスを添付してもらいReadで読む。「スクショをこのチャットに貼るかファイルパスを教えてください」と案内 |
| ダウンロードリンクを提供 | ローカル保存したファイルパスを提示 |
| 選択式UI | 使わない。自然な文章で質問する（元指示書と同じ） |

## インタビューの鉄則（全フェーズ共通）

- **1問ずつ聞く。** 複数の質問を一度に投げない（元エージェントの最重要ルール）
- 抽象的な回答には「具体的には？」「数字でいうと？」で深掘りする
- ユーザーの発言で印象的なものは原文のまま成果物に残す

## 言葉遣いの共通ルール（運営者フィードバック 2026-07-10）

- **専門用語を出力に使わない。** Who/What/How→「誰の・どんな悩みを・どう解決するか」、ペルソナ→「届けたい相手」、CTA→「行動のお願い」など、必ず日本語に言い換える
- **硬い敬語を使わない。**「させていただきます」ではなく「していきますね」。オトラボの講師が受講生に話すフレンドリーなトーン
- **数字を徹底的に集める。** 数字（kg・円・年・回数）はアカウント名・プロフィール・台本すべての精度を決める最重要素材。数字の気配がある回答には必ず「数字でいうと？」と追い質問する
- 各referenceファイル末尾の「運営者追加ルール」は原文の指示より優先する
