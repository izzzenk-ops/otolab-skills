# concept-brush-up｜コンセプト磨き上げ

## このスキルについて

インスタ初期設計ツール（insta-concept-design）で固めた発信ジャンルを土台に、「楽をしないコンセプト」理論でジャンル内の少数派ポジションを設計するスキル。流れは3ステップ：

1. **引き継ぎ** — 初期設計の成果物を読み込む
2. **10案提案** — 楽をしないコンセプト10案をHTMLレポートでChromeに表示
3. **磨き上げ** — 対話でブラッシュアップして、決定版コンセプト＋プロフィール改善案を作る

コンセプト生成の前に、必ず `references/raku-shinai-theory.md` を読むこと。理論の4ステップ・発想パターン7分類・勝ち筋の5判断材料がこのスキルのエンジン。

## Step 1：引き継ぎデータの読み込み

まず `~/Documents/Claude/Projects/インスタコンセプト設計/` 配下から対象者のフォルダを探す：

```
<対象者名または屋号>_<YYYYMMDD>/
├── 01_自己分析レポート.md   ← 強み・経験・数字の武器
├── 02_コンセプト3案.md      ← 誰の・どんな悩みを・どう解決するか（★採用案）
└── 03_プロフィール案.md     ← 現行プロフィール文
```

- フォルダが複数あれば対象者名を確認する。あるものだけ読めばよい（02だけでも動ける）
- ファイルがない場合：「初期設計ツール（Webアプリ）の結果をコピペしてもらえますか？」と案内する。Webアプリ利用者（受講生）は成果物をテキストで持っている
- それも無ければ、最低限のヒアリングをする：発信ジャンル／届けたい相手／自分の強み・実績（数字で）／売りたいもの・ゴール／続けられそうな作業量。**1問ずつ聞く**こと
- 現行プロフィール文（Instagramに今載せている文章）があるかも聞いておくと、Step 3のBefore/Afterが作れる

引き継いだら「誰の・どんな悩みを・どう解決するか」と「数字の武器」を2〜3行で要約してユーザーに見せ、ズレがないか確認してから次へ進む。

## Step 2：楽をしないコンセプト10案 → HTMLレポート

### 10案の作り方

`references/raku-shinai-theory.md` を読んだ上で、ユーザーのジャンル内で成立する「楽をしないコンセプト」を10案作る。

- **発想パターン7分類のうち最低5パターンに散らす**。10案が全部「継続実況」のような偏りは禁止。カードにパターン名をタグ表示する
- 各案は記事の型（コンセプト／伸びる理由／人に教えたくなる理由）＋**勝ち筋との相性**（この人の強み・数字・売りたいものとどう接続するか）の4点で書く
- 引き継いだ**数字の武器を必ず使う**。「−36kg」「106kg時代」のような実数がキャッチコピーに入ると強い
- 「本当にやってるの？」「自分には無理だけど見たい」「友達に送りたい」のどれかを起こせない案はボツにして作り直す
- **フォロワー0人の初日でも始められる案にする。** 受講生の多くはゼロからのスタート。フォロワーの投票・コメント頼みの参加型や、お客さん・依頼者・生徒が必要な企画は、「誰に頼めばいいの？」で初日に手が止まる。ただし企画自体を捨てるのではなく、**手が届く相手に変換する**：「人の家を片付ける」→「友人の家を片付ける」のように、友人・同僚・後輩・家族で完結する形に直すと、受講生が具体的にイメージできる強い案になる。誰を巻き込むか（友人・同僚など関係性）を必ずカードに書く
- 現実的な負荷の幅をつける：全案が「毎日3時間かかる企画」だと選べない。軽め〜重めを混ぜる
- 50コンセプト一覧は引き出しであって答えではない。ジャンルにそのまま近い案の丸写しは1〜2案まで。残りはこの人の職業・生活・性格からしか出てこない固有の案にする

### レポート生成

1. `assets/report_template.html` をベースにHTMLを生成する。デザイン（配色・フォント・角丸）はテンプレートのまま変えない。コンセプトカードを10枚、奇数は `class="concept"`・偶数は `class="concept alt"` で交互に並べる
   - ヘッダーの大見出しは**「あなただけのコンセプト案」で固定**。変えない
2. 保存先：対象者のフォルダ（無ければ `~/Documents/Claude/Projects/インスタコンセプト設計/<対象者名>_<YYYYMMDD>/` を作る）に `04_コンセプト10案.html`
3. Claude in Chrome の navigate で `file:///...` を開いてユーザーに見せる（Chrome拡張が使えない場合は `open <ファイルパス>` で開く）
4. チャットには10案のタイトル一覧だけを番号付きで書き、「気になる番号を教えてください（複数OK）」と伝える。カードの中身をチャットに再掲しない

## Step 3：対話ブラッシュアップ → 決定版

ユーザーが選んだ案（複数可）を、勝ち筋の5判断材料で磨く。**質問は1問ずつ**。

1. **続けられるか・始められるか** — 「これ、週に何回・1回何分ならできそうですか？」現実の生活に置いてみる。無理なら負荷を下げた変形案を出す。決定前に「フォロワー0の今日、最初の1本を撮れるか」も必ず確認する
2. **売りたいものにつながるか** — コンセプトが面白くても商品・サービス・ゴールへの導線が無ければ勝ち筋ではない。接続の形を一緒に設計する
3. **届けたい相手に刺さるか** — 初期設計の「誰の・どんな悩み」とズレていないか照合する

磨き方の道具：
- **掛け合わせ**：「3番の企画性×7番の続けやすさ」のように2案を合成する
- **極端化**：「もっと極端にすると？」で制限や縛りを一段強くする
- **具体化**：最初の投稿3本のイメージまで落とすと、ユーザーは「やれるかどうか」を判断できる

ユーザーが「これでいく」と決めたら、成果物を2つ保存する：

### 05_決定版コンセプト.md
- コンセプト名＋キャッチコピー
- 誰の・どんな悩みを・どう解決するか（磨き上げ後）
- 楽をしないポイント（＝何の面倒を引き受けるのか）
- 人に教えたくなる瞬間の設計
- 最初の投稿3本の案
- 売りたいものへの導線

### 06_プロフィール改善案.md
- Before：現行プロフィール（03_プロフィール案.md か、ユーザーから聞いた現行文）
- After：決定版コンセプトを反映した改善案を3案。それぞれ「何を変えたか・なぜ強くなるか」を1行添える
- 数字の武器と「楽をしないポイント」がプロフィール1行目で伝わる形にする
- Beforeが無い場合はAfter 3案のみでよい

保存したらパスを伝え、「アカウント名から作り直したい場合は insta-concept-design のPhase 3、このコンセプトでリール台本を作るなら daihon-* 系スキルへ」と次の一歩を軽く案内する。

## 言葉遣いのルール（insta-concept-design と共通）

- **専門用語を出力に使わない。** ペルソナ→「届けたい相手」、CTA→「行動のお願い」、ポジショニング→「立ち位置」など必ず日本語に言い換える
- **硬い敬語を使わない。**「させていただきます」ではなく「していきますね」。オトラボの講師が受講生に話すフレンドリーなトーン
- **数字を徹底的に集める。** 数字（kg・円・年・回数）がコンセプトとプロフィールの強さを決める。数字の気配がある回答には「数字でいうと？」と追い質問する
- レポートやチャットで理論を語りすぎない。理論はカード1枚のサマリーで十分。主役はコンセプト案
